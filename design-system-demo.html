HALFTONE_MODE

This layout uses the Container vs. Content strategy. The background is a busy pattern, but the text lives comfortably inside a solid white box.

This ensures compliance with WCAG contrast ratios while keeping the "Retro Print" vibe intact.

ARCHITECTURAL_LINES

Using hatching patterns creates depth without relying on generic drop-shadows. Combined with monospaced headers, it feels like a technical schematic.

Pattern Variants

GRID_PAPER

Blueprint/graph paper aesthetic. Perfect for technical content, data visualizations, or engineering-focused sections.

STANDARD

Clean variant without pattern. Use when you need maximum focus on content or for simpler layouts.

Usage

HTML_STRUCTURE

<article class="brutal-card-v2 brutal-card-v2--halftone">
    <div class="brutal-card-v2__header">
        <h2 class="brutal-card-v2__title">TITLE_HERE</h2>
    </div>
    <div class="brutal-card-v2__content">
        <p>Your content here...</p>
        <button class="brutal-card-v2__action">Action</button>
    </div>
</article>

Available Modifiers:

  • .brutal-card-v2--halftone (green dots)
  • .brutal-card-v2--hatching (diagonal lines)
  • .brutal-card-v2--grid (graph paper)
  • .brutal-card-v2--tabbed (folder tab header)
